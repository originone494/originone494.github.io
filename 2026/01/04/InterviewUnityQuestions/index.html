什么是“对象池”？在Unity中实现一个简单的对象池需要包含哪些核心功能？

对象池是一种性能优化设计模式，通过预先创建并复用对象实例，避免运行时频繁的Instantiate和Destroy带来的内存分配与GC开销。
核心功能包括：
初始化：预生成一定数量的对象，设为非激活状态。
获取对象：从池中返回一个可用（非激活）对象，并激活它；若无可用对象，则动态扩容（新建）。
回收对象：将使用完毕的对象设为非激活，返回到池中备用。
可选的清理功能：在场景切换或结束时，销毁池中所有对象。

Shadow Pass在视野外会被渲染吗？

不会，阴影渲染（Shadow Pass）通常基于光源的视锥体和阴影距离设置。若物体不在光源的“可见”范围内或超出阴影距离，则不会渲染其阴影，以节省性能。

动态链接库（DLL）在程序运行时被加载到内存中，可以被多个程序共享使用

[InitializeOnLoad] 是编辑器特性，仅在Unity编辑器启动或重新编译脚本后执行，打包后不会执行

Physics.Raycast 的结果与 Time.timeScale 的关系

Physics.Raycast 属于物理系统，其查询基于物理状态，不受 Time.timeScale 影响

Development Build 和 Release Build 的代码裁剪行为是完全一致的吗？

不是，Development Build 默认禁用或减少代码裁剪以便调试，而 Release Build 会进行激进的代码裁剪以减小包体

Root Motion 在 FixedUpdate 下会更精确？

是的，root Motion（根运动）通常与动画和物理交互，在 FixedUpdate 中执行能与物理引擎的固定时间步长更好同步，避免因帧率波动导致位移不精确或“抖动”。

简述 Unity 中 MonoBehaviour 脚本的生命周期（主要方法执行顺序）

  1. 初始化阶段：Awake() → OnEnable() → Start()
    Awake()：脚本实例被创建时调用一次（即使脚本未启用）。
    OnEnable()：每当对象变为激活/启用状态时调用。
    Start()：在第一次 Update 前，且脚本启用时调用一次。
  2. 更新循环阶段（每帧）：FixedUpdate() → Update() → LateUpdate()
    FixedUpdate()：按固定时间步长调用（默认0.02s），用于物理计算。
    Update()：每帧调用一次，主要的游戏逻辑入口。
    LateUpdate()：在 Update 和所有动画计算之后调用，常用于摄像机跟随。
  3. 渲染与 GUI 阶段：
    OnGUI ()：处理 GUI 绘制和事件（如按钮点击），调用次数不固定；
    OnRenderObject ()：相机渲染完所有物体后调用，适合自定义渲染逻辑。
  4. 结束/销毁阶段：
    OnDisable()：当对象变为非激活/禁用状态时调用。
    OnDestroy()：对象被销毁前调用。

请尽可能多地列举在 Unity 中索引或引用 GameObject 的不同方式

GameObject.Find(string name)：通过名称全局查找（性能差）。
Transform.Find(string path)：在特定父物体下按路径查找子物体。
GameObject.FindWithTag() / GameObject.FindGameObjectsWithTag()：通过标签查找。
GameObject.FindObjectOfType() / FindObjectsOfType()：通过组件类型查找。
GetComponent() / GetComponentInChildren() / GetComponentInParent()：获取组件，间接引用其挂载的GameObject。
序列化字段：在Inspector面板中拖拽赋值（public GameObject target; 或 [SerializeField] private GameObject target;）。
资源加载：通过 Resources.Load(path) 或 AssetBundle/Addressables 加载预制体实例化。
父子层级遍历：通过 transform.GetChild(index) 访问子物体。
发送消息/Broadcast：使用 SendMessage 或事件系统间接通信。

把struct改成class，bug消失了你怎么看？

struct在作为参数传递或赋值时是拷贝，对副本的修改不影响原始数据。改为class后，传递的是引用，修改会生效。

Unity 有几种坐标系

世界坐标系（World Space）：场景全局坐标系，物体的绝对位置；
局部坐标系（Local Space）：物体自身坐标系，子物体相对于父物体的位置；
屏幕坐标系（Screen Space）：以像素为单位，原点在左下角（UI 系统常用）；
视口坐标系（Viewport Space）：归一化坐标系（0-1 范围），适配不同分辨率。

你怎么管理游戏内存的

资源管理：AB 包（AssetBundle）分包策略（按场景 / 资源类型分包）、异步加载（避免卡顿）、无用资源及时 Unload；
对象管理：对象池（Object Pool）复用频繁创建的物体（如子弹、敌人），减少 GC；
代码层面：减少装箱拆箱、避免频繁创建临时对象、合理使用值类型。

Unity渲染管线

  1. 内置渲染管线（Built-in Render Pipeline）
    Unity 传统的默认渲染管线，属于固定功能管线（早期）+ 有限可编程管线
    优点：上手快、配置简单，无需额外安装包，适合新手和快速原型开发；对旧项目兼容性好。
    缺点：可定制化程度低，无法灵活调整渲染流程；优化空间有限，不支持光线追踪、体积雾等现代渲染特性；多平台适配时性能难以精细化控制。
    适用场景：小型独立游戏、快速 Demo 验证、老项目维护。
  2. 通用渲染管线（Universal Render Pipeline，URP）
    定位：Unity 主推的 轻量化、跨平台可编程渲染管线（SRP），前身是 LWRP（轻量级渲染管线）。
    核心特点
    优点：可高度定制渲染路径（支持前向渲染、延迟渲染），能根据项目需求开关特性；性能可控，针对移动端、低配 PC 做了深度优化；支持 SRP 合批、光照探针、阴影分级等实用功能；体积小，接入成本低。
    缺点：不支持光线追踪等高端特效，渲染上限低于 HDRP。
    适用场景：移动端手游、2D 游戏、中小型 3D 游戏（面试高频，必须结合项目）
  3. 高清渲染管线（High Definition Render Pipeline，HDRP）
    面向 主机、高性能 PC 的高端可编程渲染管线，主打次世代视觉效果。
    核心特点
    优点：支持光线追踪、体积雾、屏幕空间反射（SSR）、高动态范围（HDR）光照、次表面散射等 3A 级特效；渲染质量极致，光影表现逼真；可定制化程度最高，能满足顶级视觉需求。
    缺点：性能消耗大，对硬件要求高；不适合移动端；接入和调试成本高。
    适用场景：3A 级别主机 / PC 游戏、影视级演示项目、视觉特效（VFX）项目。
