一、什么是数据持久化

数据持久化是将内存中的数据模型转换为存储模型，以及将存储模型转换为内存中的数据模型的统称。
说人话：将游戏数据存储到硬盘，硬盘中数据读取到游戏中，也就是传统意义上的存盘。

什么是“对象池”？在Unity中实现一个简单的对象池需要包含哪些核心功能？

对象池是一种性能优化设计模式，通过预先创建并复用对象实例，避免运行时频繁的Instantiate和Destroy带来的内存分配与GC开销。
核心功能包括：
初始化：预生成一定数量的对象，设为非激活状态。
获取对象：从池中返回一个可用（非激活）对象，并激活它；若无可用对象，则动态扩容（新建）。
回收对象：将使用完毕的对象设为非激活，返回到池中备用。
可选的清理功能：在场景切换或结束时，销毁池中所有对象。

开始写作的第一步

发表于 2026-01-02 更新于 2026-01-03 分类于 写作
本文字数： 3.6k 阅读时长 ≈ 7 分钟

开书基本套路

  1. 描述主角原本的生活，让读者了解并共情你的主角
  2. 突发变故，主角原本的生活被打破，主角陷入某种困境
  3. 展示在变故中，主角为了解决突发情况，产生了什么初始目标，也就是主线
  4. 主角朝着初始目标努力的过程中，发现自己的特殊性，也就是金手指
  5. 读者产生主角运用金手指，解决当下困境的期待
  6. 作者在后文中让主角实际运用金手指，解决困境
  7. 满足读者的期待，实现爽点

身份

快速使你的角色与观众共情。

  1. 壮举
    主角初登场就靠某种夸张的行为引人注目，比如开场就被人告白，路明非的红点，遇见想要自杀的人前去阻止，刺杀完总统准备撤退等。
  2. 特殊身份
    譬如，卧底，渣男，西格玛男人，主角女装进女校，猫鼠游戏式的假身份，从没学过高达驾驶，只看一遍说明书就学会操作的高达驾驶员。
  3. 落魄的强者
    主角曾经的强大能为读者提供安全感，让其放心地阅读，多见于各种重生文。
  4. 没头脑和不高兴
    把主角和跟ta性格截然相反的角色凑在一起，比如二次元宅文里经常出现的战败文，强制爱等，他追她逃式的展开天然就吸引人。

热更新中AB包的上传和下载（一）

发表于 2025-12-31 更新于 2026-01-03 分类于 Unity
本文字数： 6.1k 阅读时长 ≈ 11 分钟

一、热更新流程

在资源需要变化时，向资源服务器（FTP、HTTP等）上传AB包和资源对比文件。
每次通过游戏客户端进入时：

  1. 下载服务器上的资源对比文件
  2. 用下载下来的资源对比文件和本地文件进行对比，记录需要进行更新的AB包
  3. 将需要更新的AB包进行更新，再更新资源对比文件
  4. 完成后，正式进入游戏

二、准备打包AB包的脚本

在github上下载AssetBundles Browser
可以通过url在包管理器里添加，也可以直接在https://github.com/Unity-Technologies/AssetBundles-Browser直接下载压缩包。
如果是直接在github上下载压缩包，需要将里面的Test文件删除，不然直接在unity里会报错
